Requirements

This Importer add-on bundle requires that you have the following software purchased separately:

  • ExpressionEngine 2.4+
  • Solspace Importer 2.2+
  • Solspace Gallery to Assets Importer
  • Pixel & Tonic Assets 2+

Install / Update

  1. Before attempting any of this, it is strongly recommended that you backup your site, and test this on a staging server.
  2. Within the download package, copy the importer.datatype.gallery folder into your sites ./system/expressionengine/third_party/importer/datatypes/ directory.
  3. Within the download package, copy the importer.content_type.assets folder into your sites ./system/expressionengine/third_party/importer/content_types/ directory.

Setup

You can import data from the ExpressionEngine 1.x Gallery module into a P&T Assets custom field in an ExpressionEngine 2.x site. It will also take into account any usage of Solspace's Gallery Extended module, which allowed one to relate Gallery Entries to Weblog Entries. The process requires a bit of set up to successfully import both the Gallery Entries and relate them to any Channel Entries. Please follow these steps:

  1. Copy all ExpressionEngine 1.x Gallery image directories over to your EE 2.x site. As each Gallery could have its own directory, insure you bring them all over.
  2. Go into these directories and delete any and all thumb or medium images. They are no longer needed.
  3. Set up these Gallery directories as ExpressionEngine Upload Directories if they are not already.
  4. Go into the Assets module and index those Upload Directories so Assets can load all of the metadata for the images.
  5. Create a new Channel for the Gallery Entries to be imported into and insure to give it an Assets custom field.
  6. If you were using the Gallery Extended module to relate gallery entries to EE 1.x Weblog entries, create an Assets custom channel field in the applicable ExpressionEngine 2.x channel for those weblog entries.
  7. As this is a complicated procedure, we recommend backing up your DB at this step.
  8. Go into Importer and set up a profile importing EE 1.x Gallery Entries into Assets. Follow the instructions and then import. During import, Importer will then go through all gallery entries and attempt to find the image in the Assets directories indexed in Step #4. If the gallery entries are related to a Weblog Entry (using Gallery Extended), it will search Expressionengine for that entry (to insure the entry from ExpressionEngine 1.x still exists in ExpressionEngine 2.x) and assign that Asset image to the Asset custom field for the related entry.
  9. In your templates, you will now have to use the Assets custom field tags for displaying your images.
  10. If you wish to have multiple image sizes, we recommended that you use one of the image resizing + caching plugins found on devot-ee.
